
# NeuroScan

**NeuroScan** is an AI-powered Brain Tumor Classification, Explainability, Segmentation, and Report Generation System developed using **PyTorch** and **Streamlit**. It is an end-to-end deep learning application that assists in the analysis of brain Magnetic Resonance Imaging (MRI) scans by combining multiple artificial intelligence techniques into a single interactive platform.

Unlike traditional classification systems that only predict the type of tumor, NeuroScan provides a comprehensive analysis workflow. After an MRI image is uploaded, the application automatically:

- Classifies the MRI into one of four categories:
  - Glioma
  - Meningioma
  - Pituitary Tumor
  - No Tumor
- Predicts the confidence score for each class.
- Generates a **Grad-CAM explainability heatmap** showing the rgfbvorms **pixel-level tumor segmentation** using a U-Net architecture to accurately localize the tumor.
- Produces a colored overlay of the predicted tumor region on the original MRI for easier visualization.
- Calculates useful segmentation statistics, including the estimated tumor area.
- Generates a **professional downloadable PDF report** containing the AI predictions, confidence scores, segmentation results, visual explanations, and medical disclaimer for documentation and review.

The project was developed as an academic prototype to demonstrate the practical application of Deep Learning, Computer Vision, and Explainable Artificial Intelligence (XAI) in medical image analysis. By combining classification, explainability, segmentation, and automated report generation into one application, NeuroScan aims to improve transparency, usability, and interpretability of AI-assisted medical imaging systems.

The system provides an intuitive interface that can be used by students, researchers, educators, and healthcare professionals for educational, research, and demonstration purposes. Although NeuroScan is capable of performing automated MRI analysis, it is intended **solely for research and educational use** and should **not** be used as a substitute for professional clinical diagnosis.

## Project Overview

NeuroScan uses two independent deep learning models working together.

### 1. EfficientNet-B0 Classifier

The classifier determines whether an MRI scan belongs to one of four classes:

* Glioma
* Meningioma
* Pituitary Tumor
* No Tumor

The classifier also outputs a confidence score for every prediction.

---

### 2. Grad-CAM Explainability

To improve transparency, NeuroScan generates a Grad-CAM heatmap highlighting the image regions that contributed most to the classification decision.

This enables users to understand **why** the model predicted a particular tumor class instead of treating the neural network as a black box.

---

### 3. U-Net Segmentation

The segmentation model performs pixel-level prediction to identify the exact tumor region.

Unlike the classifier, which predicts only the tumor type, the U-Net predicts whether each pixel belongs to the tumor, producing a binary segmentation mask that localizes the tumor within the MRI image.

NeuroScan uses two independent datasets.

Place the classification dataset inside:

```text
    ├── Training/
    │   ├── glioma/
    │   ├── meningioma/
    │   ├── pituitary/
    │   └── notumor/
    │
    │
    └── Testing/
        ├── glioma/
        ├── meningioma/
        ├── pituitary/
        └── notumor/
```

---

## Segmentation Dataset

The segmentation dataset follows a paired-image structure, where each MRI image has a corresponding binary mask with the same filename plus the `_mask` suffix.

Example:

```text
dataset/
└── Segmentation/
        ├── glioma/
        │   ├── enh_1.png
        │   ├── enh_1_mask.png
        │   ├── enh_2.png
        │   ├── enh_2_mask.png
        │   └── ...
        │
        ├── meningioma/
        ├── pituitary/
        └── ...
```

Each mask must satisfy the following conditions:

* Background pixels are black.
* Tumor pixels are white.
* Every MRI image must have exactly one corresponding mask.
* MRI images and masks must have identical filenames except for the `_mask` suffix.

Example:

```text
enh_35.png
enh_35_mask.png
```

## Running the Project

NeuroScan is organized as a Python package. Therefore, Python modules should be executed from the **project root directory** using the `-m` flag instead of running individual files directly.

### Correct

Run modules like this:

```bash
python -m segmentation.train
```

```bash
python -m segmentation.dataset
```

```bash
python -m segmentation.model
```

```bash
python -m classifier.train
```

```bash
python -m classifier.dataset
```

## To launch the Streamlit application:
In the Project root run:

```bash
streamlit run streamlit_app/app.py
```

---

### Incorrect

Avoid running modules directly like this:

```bash
python segmentation/train.py
```

```bash
python segmentation/dataset.py
```

```bash
python classifier/train.py
```

Running modules directly may result in import errors such as:

```
ModuleNotFoundError: No module named 'segmentation'
```

or

```
ModuleNotFoundError: No module named 'classifier'
```

because the project uses package-based imports.

Always execute modules using the `python -m` syntax from the project root.
